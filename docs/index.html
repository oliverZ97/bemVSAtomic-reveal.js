Probleme

  • Übersichtlichkeit
  • Wiederverwendbarkeit
  • Kaskadierung vermeiden
  • Abhängigkeit
  • tiefe Spezifität
  • kleine Dateien
  • Performance-Hürde
  • CSS ist mächtig aber oft unberechenbar

Trennung HTML und CSS

  • Kaskadierung sinkt
  • Spezifität sinkt
  • Abstraktion steigt

OOCSS

OOCSS

  • folgt nur zwei Hauptregeln
  • 1. Struktur von Aussehen trennen
  • 2. Container von Inhalt trennen

OOCSS

PRO'S:

  • Großteil wiederverwendbar
  • kleinere Dateien
  • Schnell zu laden

CON'S:

  • schnell hohe Spezifität
  • ab bestimmter Spezifität unübersichtlich

BEM

BEM

  • Namenskonvention
  • Block-Element-Modifier

BEM

PRO'S:

  • Trennung von HTML-Namen und CSS
  • übersichtliche Benennung innerhalb eines Blocks
  • Verhindert ungewollte Kaskadierung
  • Ermöglicht spezifischen Zugriff auf Element

CON'S:

  • Aufgeblähte CSS-Dateien
    • gelöst durch SCSS
    • durch Minifying
  • Kann Performance kosten

Atomic CSS

Atomic CSS

  • Namenskonvention
  • Klassen bleiben unverändert

Atomic CSS

PRO'S:

  • Keine ungewollten Funktionen
  • Wiederverwendbar
  • Kein ungenutzter Code
  • Schnell zu laden

CON'S:

  • Unübersichtlich durch Abkürzungen
  • Style-Änderungen für einzelne Elemente durchführen
    • Balance zwischen Abstraktion und Spezifität finden
  • kein spezifischer Zugriff nötig
  • viele HTML-Klassen abhängig
