BEM

PRO'S:

  • Trennung von HTML -Namen und CSS
  • übersichtliche Benennung innerhalb eines Blocks
  • Verhindert ungewollte Kaskadierung

CON'S:

  • Aufgeblähte CSS-Dateien
    • gelöst durch SCSS
    • durch Minifying
  • “hässliches” HTML
    • stört nicht den Endnutzer

Atomic CSS

Atomic CSS

  • Namenskonvention
  • Klassen bleiben unverändert

Atomic CSS

PRO'S:

  • keine ungewollten Funktionen
  • kurz aber aussagekräftig
    • wenn man die Eigenschaft ableiten kann

CON'S:

  • Abkürzungen
  • Style-Änderungen für einzelne Elemente durchführen
    • Balance zwischen Abstraktion und Spezifität finden

Fazit

  • optimale Lösung durch sinnvolle Mischung
  • zentrale Regelungen wie Farbe, Margin, Padding über Atomic
  • Komponenten Styling nach BEM
